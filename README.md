職場的Agentic時代正逐步成形

Agentic 時代的關鍵: System Design與可治理的協作規範

2024年吳恩達開始推廣AI Agent時，我原本認為這是學術界與AI廠商的事，對一般大眾影響有限。事實上目前的主流LLM如ChatGPT、Gemini、Grok、Claude等本質上都已是Agentic LLM；對一般大眾而言，只要會用瀏覽器與這些系統對話，就已經在使用agent了。

然而近期與產業界的深度交流，加上我親自體驗Antigravity+Skill開發Agent的流程後，我深刻意識到，一個由Agent自動化協作為核心的生態系(Ecosystem)將在3~5年內逐步成形。這將徹底衝擊職場中的每一個人，未來的運作模式將以 Agent-to-Agent自動化為基石，並在此之上建構Human-Agent協作機制。

近期我使用 Antigravity 結合 Skill，可以在幾分鐘內完成一個app雛型，對比幾年前用PyTorch開發Neural Network與RL，去年用n8n 建構 LLM-based workflow，這次的體驗在速度、抽象層級與產出密度上，是完全不同量級的提升，這讓我更確信由 agent 組成的生態系，將在3~5年內實質影響職場中的routine、tedious且沒有安全考量的工作。

隨者2025 MCP通訊標準開放，2026年初Skill 標準開放，以及小型推理機(如NVIDIA去年推出的Spark與AMD今年Q2要推出的Halo)開始普及，地端 Agent (Edge agent)將隨硬體普及進入日常 routine 工作。

目前多數行政秘書或知識工作者的工作型態都還是與對話型Agentic LLM 互動，僅 <2% 涉及HAI (Human-Agent Interaction)。但是隨著Skill的普及，幾年內每個職場人士都將擁有自己的Core Agents (建議<5個)。每個核心代理將管理100個由使用者以「自然語言」撰寫的專屬Skill。預計3～5年後，職場中約50%的routine、tedious、沒安全考量的工作，將由 agent-agent自動協作完成，不需 human 介入。

在邁向Agentic自動化協作的eco-system過程中，真正的關鍵在於如何設計與規範agent–agent自動化與agent–human協作，確保行為可預期、結果可解釋、責任可歸屬、系統可治理。這些問題，最終仍然回到工業工程(IE)所強調的核心能力: System Design and Problem Formulation，將需求轉為可操作、可驗證、可治理的系統結構。Agentic時代，價值不在於「會不會用 AI」，而在於「能否規範AI系統如何被使用」。
